# Example Mapping

Используется для детализации пользовательских историй через:

---

## Формат

Для каждой пользовательской истории используй структуру:

## История [номер]: [Название]

### 🟩 Правила (Rules)
Краткие утверждения, описывающие поведение системы

- Правило 1
- Правило 2

---

### 🟦 Примеры (Examples)
Конкретные сценарии (input → output)

- Пример 1:
  Дано: ...
  Когда: ...
  Тогда: ...

- Пример 2:
  Дано: ...
  Когда: ...
  Тогда: ...

---

### 🟥 Вопросы (Questions)
Неясности, пробелы, спорные моменты

- Вопрос 1
- Вопрос 2

---

## Принципы

### Правила
- Описывают поведение системы
- Короткие и однозначные
- Без технической реализации

### Примеры
- Проверяют правила
- Конкретные и проверяемые
- Желательно в формате Given/When/Then

### Вопросы
- Фиксируют неопределённости
- Помогают выявить недостающие требования

---

## Важно

- Не придумывай лишнюю функциональность
- Основывайся только на пользовательской истории
- Если информации мало — добавляй вопросы
- Каждое правило должно иметь хотя бы один пример
