# Пользовательские истории

## Формат

Пользовательские истории оформляются в следующем формате:

```markdown
## История [номер]: [Название]

**Как** [роль пользователя]  
**Я хочу** [действие или функциональность]  
**Чтобы** [цель или ценность]
```

## Структура

### Роль (Как)
- Указывает, кто является пользователем функциональности
- Примеры: владелец котика, администратор, пользователь системы

### Действие (Я хочу)
- Описывает конкретное действие или функциональность
- Должно быть конкретным и измеримым
- Избегать абстрактных формулировок

### Цель (Чтобы)
- Объясняет зачем нужна эта функциональность
- Показывает ценность для пользователя
- Связывает действие с бизнес-целью

## Примеры

**Пример 1:**
```markdown
## История 1: Напоминание о предстоящей прививке

**Как** владелец котика  
**Я хочу** получать напоминание о дате следующей прививки за 2 недели  
**Чтобы** не пропустить важную дату и подготовиться к визиту в ветеринарную клинику
```

**Пример 2:**
```markdown
## История 2: Настройка параметров напоминаний

**Как** владелец котика  
**Я хочу** выбрать за сколько времени до прививки получать напоминание и сколько напоминаний получать  
**Чтобы** настроить удобный для себя график уведомлений
```

## Принципы

1. **Одна история = одна функциональность**
2. **Фокус на пользователе**, а не на технических деталях
3. **Измеримость**: должно быть понятно, когда история выполнена
4. **Ценность**: каждая история должна приносить пользу пользователю

## Оформление файла

При создании файла с пользовательскими историями:

1. Используйте заголовок `# Пользовательские истории`
2. Разделяйте истории горизонтальной линией `---`
3. Нумеруйте истории последовательно
4. Давайте каждой истории понятное название
