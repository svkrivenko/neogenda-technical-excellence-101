# Example Mapping: История 1 — Базовый график обязательных прививок для кота

## Story
Как владелец кота  
Я хочу добавить кота с датой рождения и сразу получить график обязательных прививок  
Чтобы сразу понимать ближайшие даты вакцинации и не пропускать их

## Rules
1. Rule 1: График строится только для типа питомца `кот`.
   - Пример A: тип питомца = кот -> система переходит к построению графика.
   - Пример B (негативный): тип питомца = собака -> этот сценарий не обрабатывается в рамках истории 1.
2. Rule 2: Для расчета обязательна корректная дата рождения.
   - Пример A: дата рождения = `2025-01-10` -> данные валидны, расчет разрешен.
   - Пример B (негативный): дата рождения пустая -> расчет не выполняется, показывается ошибка "Укажите дату рождения".
   - Пример C (граничный): дата рождения = сегодня -> расчет выполняется, даты считаются от текущей даты.
3. Rule 3: В базовый график попадают только обязательные прививки из предустановленного списка.
   - Пример A: в шаблоне есть 3 обязательные прививки -> в график добавляются эти 3 прививки.
   - Пример B (негативный): в шаблоне есть обязательные и необязательные прививки -> в базовый график попадают только обязательные.
4. Rule 4: Даты прививок считаются как смещения от даты рождения.
   - Пример A: дата рождения `2025-01-10`, правило "через 8 недель" -> дата прививки `2025-03-07`.
   - Пример B: дата рождения `2025-01-10`, правило "через 12 недель" -> дата прививки `2025-04-04`.
5. Rule 5: Если обязательные данные не введены или невалидны, график не создается и пользователь получает понятную ошибку.
   - Пример A (негативный): дата рождения в будущем (`2030-01-01`) -> график не строится, ошибка "Дата рождения не может быть в будущем".
   - Пример B (негативный): дата в неверном формате -> график не строится, ошибка "Некорректный формат даты".
