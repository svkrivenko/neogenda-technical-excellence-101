# Примеры промптов

## Промпт 1. Идея продукта
``` text
Я хочу сделать трекер для прививок котам или собакам.

Опиши идею продукта в docs/idea.md
Правила - в docs/rules.md


Зафиксируй правила в файле docs/rules.md и строго соблюдай в работе.
1. Будь предельно краток, отвечай без воды.
1. Мы будем двигаться по этапам.
1. Не переходи к следующему этапу, пока не закончишь предыдущий и не получишь подтверждение, что можно двигаться дальше.
1. Двигайся маленькими шагами.
1. Не переусложняй решение, используй принципы KISS, YAGNI и бережливого подхода. Опиши эти принципы в правилах.
1. Трекай этапы и шаги в файле docs/tasklist.md.
1. Отвечай только по-русски.
1. Переспрашивай, если не уверен. Не бойся признаться, что чего-то не знаешь.
1. Не меняй никакие файлы, кроме Unit_7.

Результат этого этапа - 2 файла с идеей и правилами.
```

## Промпт 2. Этапы работы
``` text
Мы будем создавать продукт поэтапно

Этапы:
1. Описать идею продукта
2. Разложить идею продукта на пользовательские истории
3. Построить Example Mapping для 1-2 историй, с которых начнем
4. Сделать прототип в loveable.dev

Зафиксируй этапы в тасклисте.
```

## Промпт 3. Пользовательские истории
``` text
Давай начнем с 3 пользовательских историй
1. Владелец котика хочет чтобы ему напомнили про дату следующей прививки за 2 недели, чтобы ее не продолбать и подготовиться
2. Владелец котика хочет выбрать за сколько получать напоминание и сколько напоминаний
3. Владелец котика хочет быстро посмотреть какой препарат давали котику в прошый раз для подготовки к прививке

Оформи эти истории по красоте как пользовательские истории в файле docs/user_stories.md.
```
