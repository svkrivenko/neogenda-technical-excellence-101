# Промпт для Lovable

Создай веб‑приложение “Watchlist” — трекер фильмов и сериалов. Язык интерфейса: русский. Дизайн: современный, минималистичный, mobile‑first. Основной паттерн UI — карточки контента.

## Цель (обязательно учесть все истории)
Реализуй 5 историй:
1) Просмотр списка “хочу посмотреть”  
2) Добавление контента в “хочу посмотреть”  
3) Удаление контента из “хочу посмотреть”  
4) Статусы просмотра и история (“смотрю”, “посмотрено” + дата)  
5) Импорт/парсинг данных с КиноПоиска и отображение в карточках + быстрое добавление в список

## Страницы и структура
1. **Главная (“Хочу посмотреть”)**
   - Список карточек watchlist.
   - Поиск/фильтр по названию.
   - Пустое состояние: текст “Список пока пуст” + кнопка “Добавить”.
   - Каждая карточка: постер (если есть), название, тип (фильм/сериал), год (если есть), бейдж статуса (нет/“смотрю”/“посмотрено”), кнопки: “В смотрю”, “В посмотрено”, “Удалить”.
2. **Импорт с КиноПоиска**
   - Поле ввода: ссылка на страницу фильма/сериала на КиноПоиске.
   - Кнопка “Импортировать”.
   - Результат импорта — карточка‑превью (название/постер/ключевые поля).
   - Кнопка “Добавить в хочу посмотреть” (создаёт элемент watchlist).
   - Состояния: loading / error (понятный текст + “Повторить”).
3. **Детальная страница элемента**
   - Поля: постер, название, год, тип, краткое описание (если удалось импортировать), текущий статус, дата “посмотрено” (если есть).
   - Блок “История статусов”: список изменений статуса с датами (минимально: события “смотрю/посмотрено”).

## Функциональность (Acceptance ориентиры)
### Watchlist (истории 1–3)
- Просмотр: на главной всегда отображается актуальный список.
- Добавление: можно добавить вручную (минимум: название + тип) и через импорт КиноПоиска.
- Удаление: удалённый элемент исчезает из списка сразу.

### Статусы и история (история 4)
- Кнопка “В смотрю” ставит статус “смотрю” и добавляет запись в историю.
- Кнопка “В посмотрено” ставит статус “посмотрено”, сохраняет дату, добавляет запись в историю.
- Статусы видны на карточках и на детальной странице.

### Импорт/парсинг КиноПоиска (история 5)
- Для MVP реализуй “парсинг” так, чтобы прототип работал предсказуемо:
  - Если по ссылке нельзя реально получить данные из браузера из‑за CORS/ограничений, сделай адаптер‑заглушку: распознавай ID/slug из ссылки и подтягивай данные из встроенного mock‑каталога (5–8 фильмов/сериалов) с полями: `kpId`, `title`, `year`, `type`, `posterUrl`, `description`.
  - Если ссылка не распознана — показывай ошибку “Не удалось импортировать. Проверь ссылку”.
- UI должен корректно работать при неполных данных (например, без постера).

## Данные и хранение
- Храни данные в `localStorage`.
- Модель элемента watchlist (минимум):
  - `id` (uuid или kpId)
  - `source` (“manual” | “kinopoisk”)
  - `title`
  - `type` (“movie” | “series”)
  - `year` (optional)
  - `posterUrl` (optional)
  - `description` (optional)
  - `status` (“none” | “watching” | “watched”)
  - `statusHistory`: массив записей `{ status, at }`
- При первом запуске создай демо‑данные: 6 элементов (часть из КиноПоиска‑мока), чтобы были видны разные статусы и история.

## Нефункциональные требования
- Ошибки не должны “ронять” страницу: показывай сообщения и кнопку “Повторить”.
- Всё должно выглядеть как готовый прототип: аккуратная типографика, отступы, адаптивность, кнопки с понятными подписями.
